# 要件仕様書

機能要件の一覧と受け入れ条件を定義する。背景・目的は [`01-business-requirements.md`](./01-business-requirements.md)、画面仕様は [`03-functional-specification.md`](./03-functional-specification.md) を参照。

## データ項目（1 件あたり）

| 項目 | 必須 | 制約 |
|------|------|------|
| YouTube URL | ✅ | 必須 |
| タイトル | ✅ | ユーザー入力 |
| サムネ URL | — | YouTube URL からクライアント側で自動生成 |
| タグ | — | 自由入力、各 10 文字以内 |
| カテゴリ | — | 単一プリセット選択、10 文字以内 |
| 良かったポイント | — | markdown、2000 文字以内 |
| メモ | — | markdown、2000 文字以内 |
| 良かったレベル（評価） | — | 1〜5 の整数 |
| 公開日 | — | 将来拡張用、現状は未使用（nullable） |
| 追加日 | 自動 | created_at |

## 機能要件

### F-1: 一覧表示

- サムネ中心のカード表示で一覧を表示する
- カードにはサムネ・タイトル・タグ・カテゴリ・評価を表示する

### F-2: 検索

- キーワード検索（タイトル部分一致 + タグ一致）

### F-3: 並び替え

- 追加日（新しい順）
- 公開日（新しい順）※将来拡張
- 良かったレベル（高い順）

### F-4: ページング

- 一覧はページ単位で表示する
- 初期ページサイズは **10 件**
- ページ番号ボタンは最大 **5 件**を表示（現在ページ中心、端では先頭/末尾に寄せる）
- 一致件数が 0 件のときはページング UI を表示しない
- 検索・並び替え変更時は 1 ページ目へ戻る

### F-5: 管理者 CRUD（許可ユーザーのみ）

- 動画の追加・編集・削除
- 操作成功時に右上へ通知（トースト）を表示
- 入力バリデーションエラーは入力欄とメッセージで強調表示

## カテゴリプリセット

`AI` / `クラウド` / `バックエンド` / `フロントエンド` / `CI/CD` / `Linux` / `テック企業` / `プログラミング`

- `未分類`: カテゴリ未指定時のフォールバック値（UI セレクタには非表示）

## Markdown 表示要件

- 生 HTML は無効
- 出力はエスケープ
- 画像埋め込みは許可

## 受け入れ条件（抜粋）

- 必須項目（YouTube URL / タイトル）未入力時は保存されずエラー表示される
- 各上限（タグ10 / カテゴリ10 / 本文2000 / 評価1-5）を超える入力はエラーになる
- 詳細なバリデーション仕様は [`06-security-specification.md`](./06-security-specification.md)、テストケースは [`08-test-specification.md`](./08-test-specification.md) を参照
